--- layout: post title: "How to set up and run Laravel" ---

Compared to other frameworks/languages, Laravel is pretty easy to setup and run locally.

Before you try to install and run Laravel project on your local machine make sure that your setup satisfies the following requirements:

  • PHP >= 7.1.3
  • BCMath PHP Extension
  • Ctype PHP Extension
  • JSON PHP Extension
  • Mbstring PHP Extension
  • OpenSSL PHP Extension
  • PDO PHP Extension
  • Tokenizer PHP Extension
  • XML PHP Extension

The main tool used for Laravel installation is Composer - it's a PHP package manager. Make sure its installed globally on your computer. Go to https://getcomposer.org/ to download and install it if you don't have it.

There are several tools used for local development environments: XAMPP/MAMP, Valet, Homestead etc. We will utilize the simplest approach: just download Laravel via composer and run it with the help of inbuilt PHP server.

There are two ways we can install a new Laravel application. The first one:

  
  composer global require "laravel/installer"

and then:

  
  laravel new projectname

The second method:


composer create-project laravel/laravel projectname

Both commands will create a fresh directory of Laravel application. I personally prefer the first approach.

Now let's go to the freshly installed folder and check its contents:

  • app/
  • bootstrap/
  • config/
  • public/
  • resources/
  • routes/
  • storage/
  • tests/
  • vendor/
  • .editorconfig
  • .env
  • .env.example
  • .gitattributes
  • .gitignore
  • artisan
  • composer.json
  • composer.lock
  • package.json
  • phpunit.xml
  • readme.md
  • server.php
  • webpack.mix.js

Lets quickly go over those files:

app/: majority of your work will reside here (models, controllers).

bootstrap/: Laravel uses these files to start the boot procesess.

config/: a folder for configuration settings.

database/: as you already guessed, this folder contains stuff related to database, like seeds, factories and migrations.

public/: directory for static files, images, stylesheets. Also has index.php file that a web server interacts with every time someone visits your app.

resources/: files ready to be transpiled and compiled; those are mainly sass/less and javascript and also views.

routes/: route definitions live in this folder.

storage/: a place for logs, caches and compiled system files.

test/: folder for tests.

vendor/: a place where composer installs project dependencies.

.env: a file of high importance; it contains environment variables such as database credentials, app name and so on.

composer.json: defines PHP dependencies.

package.json: similar to composer.json but for front-end javascript packages.

There are other miscellaneous files in the project folder but they play a minor role. You can easily google information if the need arises.

To test our application live, type the following command into your terminal:

  
  php artisan serve

This command will spin up a local php server to help us develop and test the Laravel app.

Don't forget to initialize git repo in the project's root directory; version control is super important in modern development.
